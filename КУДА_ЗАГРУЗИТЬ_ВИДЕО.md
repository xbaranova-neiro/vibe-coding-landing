# Куда загрузить видео для мастер-класса (День 1)

Сайт использует **только свой плеер** по прямой ссылке на MP4: без перемотки, только пауза и громкость. Rutube и другие встраиваемые плееры не используются.

Видео нужно загрузить на хостинг, который отдаёт **прямую ссылку на файл** (при открытии ссылки в браузере начинается воспроизведение или скачивание MP4).

---

## Вариант 1: Яндекс Облако (Object Storage)

1. Зарегистрируйтесь: [cloud.yandex.ru](https://cloud.yandex.ru)
2. Создайте каталог → **Object Storage** → **Создать бакет**
3. Имя бакета — любое (латиница). При создании выберите **Публичный доступ: чтение объектов** (или включите его в настройках после).
4. Загрузите файл MP4 в бакет.

### Где включить доступ к файлам (если бакет уже создан)

У тебя бакет **vibe-coding** и в нём уже лежит видео. Чтобы сайт мог его открыть по ссылке:

1. В левом меню бакета нажми **«Настройки»** (иконка гаечного ключа).
2. Найди блок **«Публичный доступ»** или **«Доступ на чтение объектов»**.
3. Включи публичный доступ на чтение (галочка или переключатель «Разрешить публичное чтение объектов»).
4. Сохрани настройки.

Если такого пункта нет в «Настройках», зайди в **«Безопасность»** (иконка с людьми) и посмотри, есть ли там настройки ACL или политики доступа бакета — нужно разрешить чтение для всех (public read).

### Ссылка на твой файл

**Как получить ссылку:** в консоли Яндекс Облака открой бакет → выбери объект (файл) → в меню или панели справа нажми **«Скопировать ссылку»** / **«Получить ссылку»** и вставь её в конфиг. Так ты получишь актуальный URL, который реально отдаёт Облако.

Если составляешь ссылку вручную, формат такой (домен может отличаться — см. консоль):

```
https://storage.yandexcloud.ru/<имя-бакета>/<имя-файла>
```

Если при открытии ссылки в браузере появляется **«Этот сайт не может обеспечить безопасное подключение» (ERR_SSL_PROTOCOL_ERROR)** — значит, этот адрес/домен не поддерживает HTTPS в твоём окружении. Тогда:
- ещё раз возьми ссылку из консоли Облака (там может быть другой домен, например через CDN);
- или используй **Вариант 2: Cloudflare R2** ниже — у него стабильный HTTPS.

После включения публичного доступа вставь скопированную ссылку в **js/config.js**:

```js
window.WIPECODING_DAY1_VIDEO_MP4 = 'https://storage.yandexcloud.ru/vibe-coding/93864-vibecode_day1_2026-02-18-18-59.mp4';
```

(Имя файла должно совпадать с тем, что в бакете.)

### Если видео не воспроизводится на сайте (но открывается в новой вкладке)

Браузер блокирует запросы с другого домена (GitHub Pages, localhost), пока для бакета не настроен **CORS**.

1. В бакете **vibe-coding** открой **«Настройки»**.
2. Найди раздел **«CORS»** (или вкладку CORS). Если его нет в «Настройках», посмотри в **«Безопасность»** или в дополнительном меню бакета.
3. Добавь правило CORS (если вылезет ошибка про XML/schema — см. ниже):
   - **Allowed Origins:** один источник на строку, без запятых и лишних символов:
     ```
     https://xbaranova-neiro.github.io
     http://localhost:3333
     ```
   - **Allowed Methods:** `GET`, `HEAD` (и при возможности добавь `OPTIONS` — для предзапроса CORS).
   - **Allowed Headers:** укажи `Range` — без него браузер не сможет подгружать видео по кускам и плеер останется чёрным.
   - **Expose Headers:** укажи `Content-Range` и `Content-Length` (через запятую или с новой строки, как в форме) — иначе воспроизведение может не начаться.
   - **MaxAgeSeconds:** `0` или `3600`.
4. Сохрани. Через минуту проверь сайт снова.

**Если при сохранении CORS появляется ошибка «XML was not well-formed or did not validate»:** в поле Origins вводи только один адрес в одну строку (например, только `https://xbaranova-neiro.github.io`), без пробелов в начале и в конце. Методы — ровно `GET` и `HEAD`. Сохрани. Потом добавь второе правило с `http://localhost:3333` для локальной проверки.

### Чёрный экран — что проверить

1. **Открой ссылку на видео в новой вкладке** (та же, что в `config.js`). Если браузер показывает «Доступ запрещён» (403) или страницу с ошибкой — включи публичный доступ к бакету (см. выше «Где включить доступ»).
2. Если ссылка в новой вкладке открывается и видео идёт или скачивается — файл доступен, проблема в настройках сайта или кэше. Обнови страницу с плеером с полной перезагрузкой (Ctrl+F5).
3. Под плеером через несколько секунд может появиться красная подсказка с ссылкой «Откройте ссылку на видео в новой вкладке» — используй её для проверки.

Платно по объёму и трафику; для одного видео на 4 ГБ — обычно небольшая сумма.

## Вариант 2: Cloudflare R2

1. Зарегистрируйтесь: [dash.cloudflare.com](https://dash.cloudflare.com)
2. **R2 Object Storage** → **Create bucket**
3. Загрузите MP4 в бакет
4. Включите публичный доступ к бакету (Public access / R2.dev subdomain)
5. Скопируйте публичный URL файла
6. В **js/config.js**:
   ```js
   window.WIPECODING_DAY1_VIDEO_MP4 = 'https://ваш-адрес.r2.dev/day1.mp4';
   ```

Есть бесплатный объём; подходит для больших файлов.

## Вариант 3: Сжать видео и положить в репозиторий

Если получится ужать видео **до 100 МБ** (например, 720p, средний битрейт), файл можно положить в папку **video/** в репозитории. Тогда в конфиге укажите:

```js
window.WIPECODING_DAY1_VIDEO_MP4 = 'video/day1.mp4';
```

На GitHub Pages эта ссылка будет работать как `https://ваш-логин.github.io/vibe-coding-landing/video/day1.mp4`.

Сжать можно в [HandBrake](https://handbrake.fr/) или через FFmpeg (например, 720p, 1–2 Мбит/с).

---

## Что не подходит

- **Google Drive** — не отдаёт прямой поток для плеера, в браузере видео не воспроизводится
- **Rutube / YouTube** — встраиваемый плеер не даёт полностью отключить перемотку
- **Обычная загрузка в репо** — файлы больше 100 МБ GitHub не принимает

После того как укажете ссылку в **js/config.js**, обновите страницу — плеер подхватит новый URL.
