# Как проверить лендинг и воронку

## 1. Локально на своём компьютере

### Вариант A: через браузер по файлам
- Откройте папку `repo-landing` в проводнике (Finder).
- Дважды кликните по **index.html** — откроется лендинг в браузере.
- Проверьте: **нет редиректа** на xbaranova-neiro.ru, страница остаётся на месте.
- Нажмите любую кнопку «Зарегистрироваться» — должен открыться виджет регистрации в модальном окне.

### Вариант B: через локальный сервер (удобнее для воронки)
В терминале выполните из папки проекта:

```bash
cd "/Users/karinakomak/Вайб кодинг воронка/repo-landing"
npx serve -p 3333 .
```

Или с Python:
```bash
cd "/Users/karinakomak/Вайб кодинг воронка/repo-landing"
python3 -m http.server 3333
```

Потом откройте в браузере:
- **http://localhost:3333/** — лендинг (главная)
- **http://localhost:3333/return.html** — страница «вернитесь через 3 минуты»
- **http://localhost:3333/return.html?from_reg=1** — проверка: должен запуститься таймер 3 мин
- **http://localhost:3333/day1.html** — День 1 (после истечения 3 мин)
- **http://localhost:3333/gift.html** — тест и подарок

---

## 2. На GitHub после push

1. Запушьте изменения: `git push origin main`
2. В репозитории на GitHub: **Settings → Pages** — источник **Deploy from a branch**, ветка **main**, папка **/ (root)**.
3. Через 1–2 минуты сайт будет доступен по адресу вида:
   **https://xbaranova-neiro.github.io/vibe-coding-landing/**

Проверьте:
- **https://xbaranova-neiro.github.io/vibe-coding-landing/** — лендинг, без редиректа
- **https://xbaranova-neiro.github.io/vibe-coding-landing/return.html?from_reg=1** — воронка, таймер 3 мин

## Краткий чек-лист

| Что проверить | Ожидание |
|---------------|----------|
| Открыть index.html | Страница лендинга, **нет** перехода на neirovideo5 |
| Кнопка в шапке | Открывается виджет регистрации (модальное окно) |
| return.html?from_reg=1 | Таймер 3:00, затем **плеер на странице** и кнопка «Страница Дня 1» |
| day1.html (после 3 мин) | Плеер на странице и кнопка «Я прошёл День 1» |
| gift.html | 3 вопроса теста, после правильных ответов — ссылка на подарок |

Видео Дня 1 — прямая ссылка на MP4 в **js/config.js** (WIPECODING_DAY1_VIDEO_MP4). Инструкция: КУДА_ЗАГРУЗИТЬ_ВИДЕО.md.
